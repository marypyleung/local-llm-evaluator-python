# Local LLM Evaluator in Python 🔍
This is a transparent, cost-aware evaluation harness for RAG-generated responses. By leveraging local Hugging Face models, this evaluator ensures data privacy and allows for thousands of evaluations with zero API costs.

## 🚀 TL;DR 
A cost-effective alternative to "LLM-as-a-judge." Use local NLP models to evaluate relevance, semantic equivalence, and logical entailment via a CSV-in → CSV-out workflow. Designed for large-scale regression testing where explainability and cost matter.

## ❓ Why This Project?
<details>
<summary>Click to expand: The problem with manual and LLM evaluation</summary>
  
Traditional evaluation relies on manual review (not scalable) or LLM-as-a-judge (expensive "black box").

### In LLM training and regression testing, evaluation often starts with manual comparison of expected vs actual answers. However, in practice this approach quickly breaks down:
- **Subjectivity:** “Correct / incorrect” is often a matter of opinion among reviewers.
- **Vagueness:** Evaluation dimensions are often implicit rather than defined.
- **Scalability:** For large datasets, human review is labor-intensive and slow.

### This motivates the idea of using AI to evaluate AI to ask an LLM to act as a judge via a prompt. While this can be effective, it often introduces:
- **Cost concerns** when judging large datasets using paid LLM APIs
- A **black‑box decision process** (“the LLM decided monolithically**”)
- **Difficulty explaining** why a result failed

To code with the dilemma, this project is attempting to download free library resources and apply them for evaluating LLM output performance with reference and alignment with industry standard to provide the interested parties an other choice for affordable comparison automation.
</details>

**This project offers:**
- **Zero Cost:** Runs locally on CPU/GPU—no OpenAI/Anthropic bills.
- **Transparency:** No "black box" prompts; results come from specific NLP models (RoBERTa/MPNet).
- **Determinism:** Same input, same score. Every time.

## 📊 Evaluation Mapping with Industry Standards
Here is the summary table of what industry standard consider VS the inclusion and implemetation in our project.

| Dimension | Industry Standard | Covered in This Project? | Implementation Logic |
| :--- | :--- | :--- | :--- |
| **Relevance** | ✅ Yes | ✅ Yes | **Step 1:** Embedding Cosine Similarity (Question ↔ Actual) |
| **Semantic Correctness** | ✅ Yes | ✅ Yes | **Step 2:** Cross-Encoder (Expected ↔ Actual) |
| **Logical Correctness** | ✅ Yes | ✅ Yes | **Step 3/4:** NLI Entailment & Hallucination |
| **Faithfulness** | ✅ Yes | ⚠️ Partial | **Step 3:** Verification against user-provided atomic claims. |
| **Scope Control** | ✅ Yes | ✅ Yes | **Step 5:** NLI check ensuring response stays within Scope Claims. |
| **Hallucination** | ✅ Yes | ⚠️ Partial | **Step 6:** Heuristic Entity Extractor flags "extra" info. |
| **Fluency / Style** | ✅ Yes | ❌ No | Intentionally excluded (focus is on content correctness). |
| **Safety / Toxicity** | ✅ Yes | ❌ No | Requires specialized models (e.g., Llama Guard). |


## 🛠️ Key Design Choices
- **The "No-Claims" Solution:** Uses bidirectional NLI (Actual ↔ Expected) to check logical implication even if you haven't defined manually written atomic claims.
- **Over-generation Warnings:** Flags extra entities as `WARNING` instead of a hard failure. This keeps humans in the loop for final verification.
- **CSV-First Architecture:** Built for data pipelines. One row in, one row out. Results are stored as JSON strings in CSV columns, ready for Excel or PowerBI.
- **Local & Deterministic:** You define your own local models (e.g., `all-mpnet-base-v2`) to produce consistent scores.


## 🚫 Out of Scope
Our goal is to provide a correctness-first tool for regression testing, not to replace human qualitative review. To preserve determinism and avoid subjective evaluation, we do not evaluate:
- **Fluency/Style:** Better handled via system prompts or moderation guardrails.
- **Creativity:** Subjective quality that local models cannot reliably score.
- **Safety/Toxicity:** Requires specialized, policy-driven classifiers.

## 📂 Input Requirements
To run a batch evaluation, prepare a CSV with the following columns:
- question: The user prompt.
- expected: The "Golden Answer" reference.
- actual: The response generated by your LLM/RAG/ Chatbot.
- claims (Optional): A JSON-formatted list of facts  for Step 3 (Faithfulness) tells the evaluator exactly which facts must be present in the AI's response to pass.
  - Format: [{"claim": "The price is $50", "critical": true}]
- scope (Optional): A list of constraints to enforce for Step 5 (Scope Alignment) to ensure the LLM stays within the authorized boundaries provided in your system prompt.
  - Format: ["Do not mention competitor names", "Only discuss technical specs"]

### Evaluation Framework & Methodology
The DimensionOutcomeEvaluator suite measures performance across six critical dimensions using local NLP models:

| Steps | Evaluation Dimension | Explanation | Involved NLP/ Embedding Models |
| :--- | :--- | :--- | :--- |
| 1 |**Topic Relevance**| Ensures the LLM output directly addresses the user's question | all-mpnet-base-v2 (Cosine Similarity) |
| 2 |**Semantic Similarity** | Compares the Expected (Ground Truth) to the Actual Response for meaning-based alignment | Cross-Encoder (High-precision pairwise scoring) |
| 3 | **Faithfulness** | Verify if the response is logically supported by specific Atomic Claims | roberta-large-mnli |  
| 4 | **Logical Equivalence** | A bidirectional check to ensure the response and golden answer share the same meaning |  roberta-large-mnli  |
| 5 | **Scope Alignment** | Ensures the response adheres to specific Scope Rules (constraints) and ignores irrelevant "noise" | roberta-large-mnli  |
| 6 | **Hallucination Detection** | Identifies over-generation/extra entities not present in the reference | Heuristic Text Analysis (Non-LLM preprocessing) |

You can change the models/ remove any parts of the steps based on your needs.

```python
class DimensionOutcomeEvaluator:
    """
    A comprehensive evaluation suite for LLM responses. 
    It measures performance across five dimensions: Relevance, Similarity, 
    Faithfulness (NLI), Scope Alignment, and Over-generation.
    """


    def __init__(
        self,
        embed_model = "sentence-transformers/all-mpnet-base-v2", # General-purpose embedding model for semantic search
        cross_encoder_model = "cross-encoder/stsb-roberta-large", # High-accuracy model for re-ranking and similarity
        nli_model = "roberta-large-mnli", # Inference model to check if 'A entails B' or 'A contradicts B'
        use_cross_encoder = True, # Set to False to skip heavy Cross-Encoder computation
        device = -1,   # Use -1 for CPU; use 0 for GPU (recommended for large datasets)
    ):
        """Initializes AI models for embedding, cross-encoding, and Natural Language Inference (NLI)."""
        self.embedder = SentenceTransformer(embed_model)
        self.use_cross_encoder = use_cross_encoder
        self.cross_encoder = CrossEncoder(cross_encoder_model) if use_cross_encoder else None

        # The NLI pipeline determines the logical relationship between two sentences
        self.nli = pipeline(
            "text-classification",
            model=nli_model,
            top_k=None,
            truncation=True,
            device=device
        )

    # -------------------------
    # 1) Topic Relevance (Question vs. Actual Response)
    # -------------------------
    def topic_relevance(self, question, actual):
        """
        Measures how well the LLM response stayed on topic.
        Uses Cosine Similarity to compare the user's question with the AI's response.
        
        Returns:
            - PASS: High alignment (>0.65)
            - BORDERLINE: Weak alignment (0.50-0.65)
            - FAIL: Irrelevant response (<0.50)
        """
        q = "" if question is None else str(question)
        a = "" if actual is None else str(actual)

        q_emb = self.embedder.encode(q, normalize_embeddings=True)
        a_emb = self.embedder.encode(a, normalize_embeddings=True)
        sim = float(util.cos_sim(q_emb, a_emb).item())

        if sim >= 0.65:
            status = "PASS"
        elif sim >= 0.50:
            status = "BORDERLINE"
        else:
            status = "FAIL"

        return {"status": status, "similarity": round(sim, 3)}

    # -------------------------
    # 2) Semantic Similarity (Expected vs. Actual Response)
    # -------------------------
    def semantic_similarity(self, expected, actual):
        """
        Uses a Cross-Encoder to compare the AI's response against a 'Ground Truth' answer.
        Cross-Encoders are more computationally demanding but far more accurate than standard embeddings.
        """
        if not self.use_cross_encoder:
            return {"level": "SKIPPED", "similarity": None}

        exp = "" if expected is None else str(expected)
        act = "" if actual is None else str(actual)
        
        # Normalize scores from standard 0-5 scale to a 0-1 scale
        raw = float(self.cross_encoder.predict([(exp, act)])[0])
        sim = raw / 5.0 if raw > 1.0 else raw
        sim = max(0.0, min(1.0, sim))

        if sim >= 0.75:
            level = "HIGH"
        elif sim >= 0.55:
            level = "MEDIUM"
        else:
            level = "LOW"

        return {"level": level, "similarity": round(sim, 3)}

    # -------------------------
    # NLI Helper Methods for Step 3, 4 & 5(Internal Logic)
    # -------------------------
    
    def _normalize_nli_output(self, out):
        """Preprocessing to standardizes HuggingFace's varying output formats into a consistent list."""
        if isinstance(out, dict): return [out]
        if isinstance(out, list) and len(out) > 0:
            return out if isinstance(out[0], dict) else out[0]
        return []

    def _nli(self, premise, hypothesis):
        """
        The project's core 'Logical Brain.' 
        Powers Step 3 (Entailment), Step 4 (Hallucination) and Step 5 (Scope) by checking if 
        the LLM response (Premise) supports or contradicts a Claim (Hypothesis).
        
        Uses RoBERTa-specific '</s></s>' formatting for dual-sentence processing.
        """
        premise = "" if premise is None else str(premise)
        hypothesis = "" if hypothesis is None else str(hypothesis)

        # Standard RoBERTa separator format
        text = f"{premise} </s></s> {hypothesis}"
        out_raw = self.nli(text)
        out = self._normalize_nli_output(out_raw)

        scores = {i.get("label", "").upper(): float(i.get("score", 0.0)) for i in out if i.get("label")}
        label = max(scores, key=scores.get) if scores else "UNKNOWN"
        conf = scores.get(label, 0.0)

        return {
            "label": label,
            "confidence": round(conf, 3),
            "scores": {k: round(v, 3) for k, v in scores.items()}
        }

    # -------------------------
    # 3) Entailment Outcome (Fact-Checking)
    # -------------------------
    def entailment_outcome(self, actual, claims):
        """
        Verifies the LLM response against a list of specific factual claims.
        If a 'critical' claim is contradicted, the entire test fails.
        
        Ideal for: Ensuring the LLM doesn't lie about specific product details or policies.
        """
        actual = "" if actual is None else str(actual)
        if not claims:
            return {"overall_status": "SKIPPED", "per_claim": []}

        per_claim, total_crit, entailed_crit, contra_crit = [], 0, 0, 0

        for c in claims:
            claim_text = str(c.get("claim", "")).strip()
            is_critical = bool(c.get("critical", True))
            if not claim_text: continue

            r = self._nli(actual, claim_text)
            per_claim.append({"claim": claim_text, "critical": is_critical, **r})

            if is_critical:
                total_crit += 1
                if r["label"] == "ENTAILMENT": entailed_crit += 1
                if r["label"] == "CONTRADICTION": contra_crit += 1

        # Logic: Contradicting a critical fact is an automatic FAIL
        if contra_crit > 0:
            overall = "FAIL"
        elif total_crit > 0 and entailed_crit == total_crit:
            overall = "PASS"
        else:
            overall = "PARTIAL"

        return {
            "overall_status": overall,
            "entailed_critical": entailed_crit,
            "total_critical": total_crit,
            "contradicted_critical": contra_crit,
            "per_claim": per_claim
        }

    # -------------------------
    # 4) Hallucination Indicator (Bidirectional Check)
    # -------------------------
    def sentence_entailment_indicator(self, expected, actual, conf_pass = 0.60):
        """
        Checks if the 'Actual' and 'Expected' answers support each other.
        - actual_entails_expected: Checks for missing info (Under-generation).
        - expected_entails_actual: Checks for hallucinations (Over-generation/False info).
        """
        exp = "" if expected is None else str(expected)
        act = "" if actual is None else str(actual)

        a_to_e = self._nli(act, exp)
        e_to_a = self._nli(exp, act)

        if a_to_e["label"] == "ENTAILMENT" and a_to_e["confidence"] >= conf_pass:
            status = "PASS"
        elif a_to_e["label"] == "CONTRADICTION":
            status = "FAIL"
        else:
            status = "REVIEW"

        return {
            "status": status,
            "actual_entails_expected": a_to_e,
            "expected_entails_actual": e_to_a
        }

    # -------------------------
    # 5) Scope Alignment (Constraint Checking)
    # -------------------------
    def scope_alignment_outcome(self, actual, scope_claims):
        """
        Ensures the model adheres to specific 'Scope' rules (e.g., "Must not mention competitors").
        The status is 'PASS' only if EVERY scope claim is followed.
        """
        actual = "" if actual is None else str(actual)
        if not scope_claims:
            return {"status": "SKIPPED", "checks": []}

        checks = []
        for sc in scope_claims:
            r = self._nli(actual, str(sc).strip())
            checks.append({"scope_claim": sc, **r})

        passed = all(c["label"] == "ENTAILMENT" for c in checks)
        return {"status": "PASS" if passed else "FAIL", "checks": checks}

    # -------------------------
    # 6) Over-generation (Extra Information Detection)
    # -------------------------
    def _extract_entities_heuristic(self, text):
        """Helper to find proper nouns, acronyms, and bracketed text as proxies for 'facts'."""
        entities = set()
        for m in re.findall(r"\b[A-Z]{2,}\b", text or ""): entities.add(m) # Acronyms
        for m in re.findall(r"\(([^)]+)\)", text or ""): entities.add(m.strip()) # Parentheses
        for m in re.findall(r"\b([A-Z][a-z]+(?:\s+[A-Z][a-z]+){1,6})\b", text or ""): entities.add(m.strip()) # Names
        return sorted(entities)

    def overgeneration_outcome(self, expected, actual):
        """
        Flags cases where the LLM model provides too much information.
        It compares 'entities' (Names, Acronyms) in the actual answer vs. the expected answer.
        If the AI introduces too many new entities, it triggers a WARNING.
        """
        exp_ents = set(self._extract_entities_heuristic(expected))
        act_ents = set(self._extract_entities_heuristic(actual))
        extra = sorted(list(act_ents - exp_ents))

        if len(exp_ents) == 0:
            status = "OK" if len(extra) <= 3 else "WARNING"
        else:
            ratio = len(extra) / max(1, len(exp_ents))
            status = "OK" if ratio <= 1.0 else "WARNING"

        return {
            "status": status,
            "extra_entities": extra,
            "count_extra": len(extra)
        }
```

### Batch Evaluation Wrapper `evaluate_lists` script

```python
def evaluate_lists(
    question_list,
    expected_list, 
    actual_list, 
    claims_list=None,
    scope_claims_list=None,
    use_cross_encoder=True):
    """
    Orchestrates batch evaluation of LLM responses across six dimensions.
    Returns: List[Dict] containing the icon-coded reports for each response pair.
    """

    # Initialize evaluator once to prevent reloading models (VRAM efficient)
    evaluator = DimensionOutcomeEvaluator(use_cross_encoder=use_cross_encoder)
    results = []
    n = len(expected_list)

    for i in range(n):
        q = question_list[i] if question_list else ""
        exp = expected_list[i]
        act = actual_list[i]
        row_result = {}

        # 1. 💬 Topic Relevance
        row_result["topic_relevance"] = evaluator.topic_relevance(q, act)
        
        # 2. 🎯 Semantic Similarity
        row_result["semantic_similarity"] = evaluator.semantic_similarity(exp, act)

        # 3. 📜 Faithfulness (NLI-based Fact Checking)
        if claims_list is not None and i < len(claims_list) and claims_list[i]:
            row_result["entailment"] = evaluator.entailment_outcome(act, claims_list[i])
        else:
            row_result["entailment"] = {"overall_status": "SKIPPED", "reason": "No claims provided"}

        # 4. ⚖️ Logical Equivalence (Bidirectional Entailment)
        row_result["logical_equivalence"] = evaluator.sentence_entailment_indicator(exp, act)

        # 5. 🛡️ Scope Alignment (Constraint Enforcement)
        if scope_claims_list is not None and i < len(scope_claims_list) and scope_claims_list[i]:
            # FIX: Ensure the scope claims are passed to the evaluator
            row_result["scope_alignment"] = evaluator.scope_alignment_outcome(act, scope_claims_list[i])
        else:
            row_result["scope_alignment"] = {"status": "UNKNOWN", "reason": "No scope criteria defined"}

        # 6. ⚠️ Over-generation (Hallucination/Heuristic Analysis)
        row_result["over_generation"] = evaluator.overgeneration_outcome(exp, act)

        results.append(row_result)

    return results
```
